# 14 天内容日历（开源 + 小红书/抖音 + SEO）

目标：用最小成本在 14 天内做到：
- 1 个开源仓库上线
- 10 条短视频发布
- 3 篇 SEO 文章发布
- 收集 30 个 waitlist 邮箱或 10 个有效对话

产品：Changelog-to-Impact Copilot（Stripe + OpenAI）

---

## Day 1
- GitHub：发 repo（哪怕只有 README + roadmap + 示例输出）
- 视频：#1《Stripe 更新又改了？10 秒定位受影响代码》
- SEO：确定关键词与文章标题（Stripe breaking changes / OpenAI deprecation）

## Day 2
- GitHub：补一个“示例报告文件（reports/sample.md）”
- 视频：#2《自动生成升级 Issue》
- 社区：发到 1 个开发者群/朋友圈/同学群，求 star 和反馈

## Day 3
- GitHub：加“配置文件模板 changelog-impact.yaml 示例”
- 视频：#3《Dependabot 不够：API 行为变更才致命》

## Day 4
- SEO：文章 1 发布
  - 标题：Stripe breaking changes: how to detect impact in your code
- 视频：#4《OpenAI 模型弃用前，我怎么提前知道》

## Day 5
- GitHub：加“使用截图/GIF”（放到 README）
- 视频：#5《升级不是难，难在找影响点》

## Day 6
- 小红书：发图文《小团队如何避免第三方 API 更新导致事故》
- 视频：#6《我做了个开源 CLI：输入 repo 输出影响点》

## Day 7
- GitHub：发一个 Issue 模板（.github/ISSUE_TEMPLATE/upgrade.md）
- 视频：#7《Stripe API 升级 checklist（送模板）》

## Day 8
- SEO：文章 2 发布
  - 标题：OpenAI model deprecations: how to avoid production outages
- 视频：#8《把 changelog 变成 PR：我正在做迁移机器人》

## Day 9
- 外呼：给 20 个潜在 ICP 发私信（LinkedIn/邮件/微信）
- 视频：#9《做 SaaS 最怕什么？不是 bug，是外部依赖变了》

## Day 10
- GitHub：发布 v0.1.0 tag（哪怕功能很少，也要“可运行”）
- 视频：#10《开源 + SaaS 怎么组合变现》

## Day 11
- SEO：文章 3 发布
  - 标题：Dependabot is not enough: API behavior changes vs package updates
- 社区：发到 Reddit/HN/掘金（看你渠道偏好）

## Day 12
- 小红书：发图文《我为什么选 Stripe+OpenAI 这个微小但刚需的工具》
- 收集反馈：整理 10 条反馈进入 roadmap

## Day 13
- 外呼：预约 3 个 15 分钟访谈
- 准备“托管版 beta”报名表（Notion/Google Form）

## Day 14
- 复盘：
  - 哪条视频带来最多关注
  - 哪篇文章带来最多访问
  - 哪类用户最愿意聊
- 决策：
  - 继续加功能 or 开始做托管版
